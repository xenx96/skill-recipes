# OSS Code-Level Feature Analysis (OSS 소스 코드 분석)

오픈소스 GitHub 레포지토리의 소스 코드를 직접 읽어
특정 기능이 어떻게 구현되었는지 분석합니다.

이 스킬은:

- 두 가지 모드 지원: **Compare** (프로젝트 간 비교), **Deep Dive** (단일 프로젝트 심층 분석)
- GitHub 소스 트리를 탐색하여 디렉토리 구조와 핵심 모듈을 매핑
- 실제 소스 코드를 읽어 아키텍처, 데이터 구조, 알고리즘, 에러 처리를 분석
- 확장 포인트와 기술 스택을 식별
- 비교표 (compare 모드) 또는 아키텍처 다이어그램 (deep-dive 모드) 생성
- 트레이드오프 분석과 함께 전략적 권고안을 제공

---

## 왜 필요한가

문서를 읽으면 라이브러리가 무엇을 하는지 알 수 있습니다.
소스 코드를 읽으면 어떻게, 그리고 왜 그렇게 했는지 알 수 있습니다.
이 스킬은 성숙한 OSS 프로젝트에서 실행 가능한 구현 인사이트를 추출합니다:
문서에서는 거의 다루지 않는 아키텍처 결정, 검증된 패턴, 트레이드오프를 파악합니다.

---

## 분석 모드

### Compare

동일 기능을 2~5개 OSS 프로젝트에서 비교 분석합니다.
아키텍처, 데이터 모델, 알고리즘, 에러 처리, 확장성, 복잡도에 걸쳐
구조화된 비교표를 생성합니다.

### Deep Dive

단일 프로젝트의 기능 구현을 심층 분석합니다.
아키텍처 다이어그램 (Mermaid), 데이터 플로우 다이어그램,
호출 체인, 설계 결정 분석을 생성합니다.
