# OSS Code-Level Feature Analysis

Explore open-source GitHub repositories at the source code level
to understand how specific features are implemented.

This skill:

- Supports two modes: **Compare** (cross-project) and **Deep Dive** (single project)
- Browses GitHub source trees to map directory structures and key modules
- Reads actual source code to analyze architecture, data structures, algorithms, and error handling
- Identifies extension points and technology stacks
- Produces comparison tables (compare mode) or architecture diagrams (deep-dive mode)
- Provides strategic recommendations with trade-off analysis

---

## Why

Reading documentation tells you what a library does.
Reading source code tells you how — and why.
This skill extracts actionable implementation insights from mature OSS projects:
architectural decisions, proven patterns, and trade-offs that documentation rarely covers.

---

## Analysis Modes

### Compare

Analyze the same feature across 2–5 OSS projects.
Produces a structured comparison table across architecture, data model,
algorithms, error handling, extensibility, and complexity.

### Deep Dive

Deeply analyze a single project's feature implementation.
Produces architecture diagrams (Mermaid), data flow diagrams,
call chains, and design decision analysis.
