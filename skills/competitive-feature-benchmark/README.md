# Competitive Feature Benchmark

Research and compare how competing products implement a similar feature
at the UX and interaction level.

This skill:

- Identifies 3–5 relevant competitors in the same domain
- Analyzes information architecture, navigation, scalability, power-user support, and edge case handling per competitor
- Produces a structured comparison table
- Synthesizes common patterns, outliers, and anti-patterns
- Provides strategic differentiation recommendations

---

## Why

Feature design decisions should be evidence-based, not opinion-driven.
Understanding how competitors solve the same problem reveals proven patterns,
avoidable pitfalls, and opportunities to differentiate — without blind imitation.

---

## Analysis Dimensions

- **Information Architecture** — flat vs hierarchical, grouping, context preservation
- **Navigation & Interaction** — click depth, progressive disclosure, control types
- **Scalability Handling** — large datasets, search/filter/sort, lazy loading
- **Power-user Support** — bulk actions, keyboard shortcuts, customization
- **Edge Case Handling** — empty states, permissions, long labels, deep nesting
