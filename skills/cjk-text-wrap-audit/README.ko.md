# CJK 텍스트 줄바꿈 감사 (CJK Text Wrap Audit)

웹 UI에서 발생하는 한국어·일본어·중국어(CJK) 줄바꿈 버그를 계층화된
증거 우선(evidence-first) 절차로 진단하고 수정한다. 코드를 고치기
**전에** 진단·검증 보고서를 먼저 산출한다.

이 스킬은 다음을 수행한다:

- 증상을 정확히 포착(로케일, 브레이크포인트, 줌, 스크린샷)
- 문제 번역 문자열을 유니코드 코드포인트로 덤프하여 숨겨진
  NBSP / ZWJ / soft hyphen을 드러낸다
- `word-break` / `overflow-wrap` / `text-wrap` 캐스케이드를 추적해
  *실제로 적용된* 선언을 찾아낸다
- 전역 규칙을 덮어쓰는 UI 라이브러리 인젝터(Ant Design `cssinjs`,
  MUI, Mantine, Chakra 등)를 식별한다
- **최소 개수**의 수정 레이어를 제안한다 — 전역 캐스케이드 →
  라이브러리 오버라이드 → 헤드라인 밸런싱 → 번역 NBSP
- 줌 × 브레이크포인트 × 로케일 × 테마 매트릭스로 수정 결과를 검증한다

---

## 왜 필요한가

CJK 줄바꿈 버그는 캐스케이드 결함의 한 부류다. `body`에 빠진
`word-break` 규칙, 명세도(specificity) 싸움에서 이기는 UI 라이브러리의
`cssinjs` 규칙, 과도하게 작동하는 `text-wrap` 알고리즘, 어절 경계가
없는 번역 문자열 — 이 네 가지 중 어느 하나만으로도 한 음절 한가운데서
글자가 쪼개지거나 한 글자가 줄 끝에 외톨이로 남는 사고가 발생한다.

`<br>` 태그, `!important`, 하드코딩된 `max-width`, 또는 명세도가 0이
되어버리는 `:where()` 오버라이드 같은 임시방편은 다음 엔지니어가
같은 문제를 다시 발견하게 만든다. 이 스킬은 반복되는 디버깅 여정을
고정된 게이트 절차로 바꿔, 추측 대신 **증거에 기반한 보고서**를
만들어낸다.

---

## 절차 (7개의 게이트)

0. **증상 포착(Symptom Capture)** — 스크린샷, URL, 로케일,
   브레이크포인트, 브라우저, 줌 레벨을 수집한다. 하나라도 빠지면 중단.
1. **원시 문자열 분석(Raw String Forensics)** — 번역 문자열을 유니코드
   코드포인트로 덤프하고 어절 경계를 표시한다.
2. **계산된 스타일 검사(Computed Style Inspection)** — `word-break`,
   `overflow-wrap`, `text-wrap`, `hyphens`, `line-break` 캐스케이드를
   추적해 *실제로 적용된* 선언과 그 명세도를 기록한다.
3. **오버라이드 추적(Override Tracing)** — UI 라이브러리 `cssinjs`
   인젝터를 식별하고, 안전한 오버라이드 위치를 결정한다(이중 클래스
   기법 사용, `:where()` 금지, `!important` 금지).
4. **계층적 수정 제안(Layered Fix Proposal)** — 필요한 **최소 수의
   레이어**만 제안한다: 전역 캐스케이드 → 라이브러리 오버라이드 →
   헤드라인 `text-balance` → 번역 NBSP. 첫 번째 레이어로 증상이
   해결되면 거기서 멈춘다.
5. **검증 매트릭스(Verification Matrix)** — 줌 × 브레이크포인트 ×
   로케일 × 테마로 재검증한다. **모든 셀이 통과**할 때만 수정 완료
   (라틴 로케일 컨트롤 포함).
6. **승인 게이트(Approval Gate)** — 보고서를 제시하고 사용자의 명시적
   승인을 받기 전까지는 코드를 수정하지 않는다.

---

## 핵심 가드레일

- **`!important` 금지.** 명세도 싸움은 `.foo.foo` 이중 클래스 기법으로
  이긴다.
- **`:where(...)` 금지.** 명세도가 0,0,0으로 떨어져 라이브러리 규칙에
  소리 없이 진다.
- **`word-break: keep-all`을 `code` / `pre` / `kbd` / `samp`에 적용
  금지.** 코드 블록 렌더링이 망가진다.
- **NBSP는 짧은(≤4단어) 의미 단위에만.** 길게 쓰면 줄바꿈이 아예
  되지 않아 좁은 화면에서 가로 오버플로가 발생한다.
- **150% 이상의 브라우저 줌에서 반드시 검증.** WCAG 1.4.4 (텍스트
  크기 조정) 요구사항이며, CJK 줄바꿈 버그는 줌을 올렸을 때만 드러나는
  경우가 많다.
- **출시 중인 모든 CJK 로케일에서 검증.** `ko`만이 아니라 `ja`, `zh`도
  같은 문자열이 다르게 줄바꿈된다.
- **라틴 로케일 회귀 없음을 확인.** 독일어는 영어보다 약 30% 길어서
  헤드라인 줄바꿈의 표준 회귀 케이스다.
