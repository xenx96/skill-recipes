# Change Re-Audit (변경 재감사)

코드 변경사항을 재감사하여 사이드 이펙트, 회귀 위험,
미처리 엣지 케이스를 머지 또는 배포 전에 식별합니다.

이 스킬은:

- 변경사항을 논리적 Change Unit으로 분해
- 7개 리스크 차원에서 사이드 이펙트를 평가
- 엣지 케이스 검증 매트릭스를 구축
- 회귀 갭을 식별하고 최소한의 테스트를 제안
- 발견 사항을 P0 / P1 / P2로 분류하며 file:line 근거 제시

---

## 왜 필요한가

일반적인 코드 리뷰는 명확한 문제를 잡아냅니다.
하지만 사이드 이펙트, 숨겨진 회귀, 엣지 케이스는
복잡한 리팩토링이나 다중 모듈 변경에서 자주 빠져나갑니다.
이 스킬은 구조화된 리스크 분석을 통한 체계적인 2차 점검을 제공합니다.

---

## 절차 (5-Gate)

0. **Change Mapping** — diff를 Change Unit으로 분해
1. **Side Effect Analysis** — 공개 계약, 상태 변이, 동시성, 리소스, 보안, 관측성, 운영 리스크
2. **Edge Case Matrix** — null/empty, 경계값, 대량 입력, 부분 실패, 중복, 권한, 하위 호환성
3. **Regression Proof** — 커버된 시나리오 vs 미커버 영역, 최소 테스트 제안
4. **Code Hygiene Check** — 데드 코드, 불필요한 추상화, 사일런트 드리프트
