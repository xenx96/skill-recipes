# Action Validation Planner

분석이나 설계 결과를 Codex, Claude Code, Cursor 같은 CLI 코딩 에이전트가 바로 실행할 수 있는 작업 지시서로 변환하는 스킬입니다.

이 스킬은 특히 **성능이 중요한 백엔드/데이터 엔지니어링 작업**에 맞춰져 있습니다. 설명에서 끝나지 않고, 실제 구현·검증·성능 확인·롤백까지 포함한 실행 가능한 계획을 생성합니다.

포함 내용:

- 구현 범위
- 최소 변경 중심 Action Plan
- 검증 명령어
- 성능 측정 방법
- 결과 정확성 검증
- 실패 케이스
- 롤백 계획
- Codex에 그대로 붙여 넣을 수 있는 실행 프롬프트

---

## 왜 필요한가

Claude는 설계와 판단에 강하지만, 그 결과를 Codex에 그대로 넘기면 다음 정보가 부족한 경우가 많습니다.

- 어떤 파일을 먼저 봐야 하는지
- 어떤 파일만 수정해야 하는지
- 절대 건드리면 안 되는 파일은 무엇인지
- 무엇을 실행해야 성공/실패를 판단할 수 있는지
- 성능 개선 여부를 어떻게 측정할지
- 실패하면 어떻게 되돌릴지

이 스킬은 설계 결과를 **범위가 명확하고, 검증 가능하며, 성능을 고려한 실행 지시서**로 바꿉니다.

---

## 적합한 작업

다음 작업에 적합합니다.

- 백엔드/데이터 엔지니어링 변경
- 쿼리 최적화
- ETL/데이터 파이프라인 변경
- ClickHouse, Delta Lake, DDS, Kafka, WASM UDF, Docker, Kubernetes 작업
- 회귀 위험이 있는 리팩토링
- Claude → Codex 핸드오프
- 성능, 정확성, 롤백이 중요한 작업

---

## 사용하지 말아야 할 경우

다음 상황에는 사용하지 않는 것이 좋습니다.

- 순수 개념 설명
- 구현 대상이 없는 브레인스토밍
- 한 줄 수정 수준의 단순 작업
- 비기술적 글쓰기
- 저장소, 명령어, 검증 경로가 전혀 없는 작업

---

## 출력 원칙

출력은 다음 조건을 만족해야 합니다.

1. **실행 가능성** — 다른 CLI 에이전트가 바로 작업할 수 있어야 함
2. **범위 제한** — 수정 범위와 제외 범위가 명확해야 함
3. **검증 가능성** — 정확성/성능 검증 방법이 있어야 함
4. **토큰 효율성** — 배경 설명을 반복하지 않고 필요한 정보만 제공
5. **안전성** — 파괴적 명령, 시크릿, 무관한 파일 접근 방지

## 사용 예시

사용자 요청:

> Claude가 만든 설계를 Codex가 바로 실행할 수 있는 구현/검증 계획으로 바꿔줘.

스킬 출력:

- 먼저 확인할 파일
- 수정 가능성이 높은 파일
- 단계별 구현 순서
- 테스트/빌드/쿼리 실행 명령어
- 성능 측정 명령어
- 롤백 명령어
- Codex에 붙여 넣을 최종 프롬프트
