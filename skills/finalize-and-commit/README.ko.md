# Finalize Changes and Commit (변경사항 최종 정리 및 커밋)

코드 변경사항을 프로덕션 수준으로 최종 정리합니다.
중복 로직 제거, 하드코딩 값 감사, 빌드 무결성 검증,
Conventional Commits 형식의 깔끔한 커밋 구조화를 수행합니다.

이 스킬은:

- 현재 세션 변경사항을 격리하고 범위 외 파일을 보호
- 중복 로직과 데드 코드를 탐지 및 제거
- 하드코딩 값 감사 (알고리즘 상수, 운영 정책, 테스트 전용)
- 일관성, 보안, 빌드 무결성 검증
- 논리적 커밋 분리: 리팩토링 → 기능 변경 → 테스트/문서

---

## 왜 필요한가

정리되지 않은 작업 변경사항은 지저분한 커밋 히스토리를 만듭니다.
하드코딩 값, 중복 로직, 데드 코드는 체계적인 최종 점검 없이는
프로덕션에 그대로 들어갑니다. 이 스킬은 다른 세션의 변경사항을
실수로 스테이징하지 않으면서 모든 커밋이 깔끔하고 의도적이며
안전하도록 보장합니다.

---

## 절차 (6-Gate)

0. **Working Set Validation** — 세션 변경 격리, 범위 외 파일 보호
1. **Duplicate & Dead Code Detection** — 3회 이상 반복 시 헬퍼 추출, 미사용 코드 제거
2. **Hardcoded Value Audit** — 알고리즘 상수 / 운영 정책 / 테스트 전용으로 분류
3. **Consistency & Quality Review** — 에러 처리, 로깅, 시크릿, 인터페이스 호환성
4. **Verification Proof** — 테스트, 린트, 타입체크, 빌드 실행
5. **Commit Structuring** — 유형별 분리, Conventional Commits 형식
